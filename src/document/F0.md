
# 基频 （`Fundamental Frequency, F0`）

基频是指声带振动的基本频率，单位通常是赫兹(`Hz`),对应感知上的"音高"(`pitch`)。语音信号分析中最基础且重要的特征之一。

## 物理意义

- 反映声带的振动特性
- 与说话人的性别、年龄、情绪状态相关
- 是语调(`intonation`)和声调(`tone`)的物理基础

## 基频相关特征

### 原始基频轨迹：

- 随时间变化的基频值序列
- 反映语调变化

### 统计特征：

- 平均基频(`mean F0`)
- 基频标准差(`F0 standard deviation`)
- 最大/最小基频
- 基频范围(`F0 range`)

### 动态特征：

- 基频变化率(`ΔF0`)
- 基频加速度(`ΔΔF0`)
- 基频轮廓斜率

### 归一化特征：

- 对数基频(`log F0`)
- `z-score`归一化基频
- 说话人归一化基频

## 基频提取方法

### 时域方法：
#### 自相关法(`Autocorrelation`): 通过计算信号的自相关性寻找周期性，峰值位置对应基频周期。

  - 优点：计算简单，实时性好。  
  - 缺点：对噪声敏感。

#### 平均幅度差函数(`AMDF`):通过计算信号与延迟版本的差值的平均值来检测周期性。

  - 优点：计算量低。  
  - 缺点：对高频噪声敏感。

#### 过零率(`Zero Crossing Rate`)

#### `YIN`算法(改进的自相关方法，通过差分函数消除倍频误差)：适用低信噪比环境

**核心思想**：改进的自相关方法，通过差分函数解决传统自相关的倍频/半频错误问题。  

**关键步骤**：  

1. **差分函数（Difference Function）**：  
   计算信号与延迟版本的平方差，替代自相关，减少峰值误判。  
   \[
   d_t(\tau) = \sum_{j=1}^{W} (x_j - x_{j+\tau})^2
   \]  
   - \(W\)：分析窗长，\(\tau\)：延迟时间。  
2. **累积均值归一化（Cumulative Mean Normalization）**：  
   对差分函数进行归一化，消除幅度影响：  
   \[
   d_t'(\tau) = \begin{cases} 
   1 & \text{if } \tau = 0 \\
   d_t(\tau) / \left( \frac{1}{\tau} \sum_{j=1}^{\tau} d_t(j) \right) & \text{otherwise}
   \end{cases}
   \]  
3. **峰值检测**：  
   寻找归一化差分函数的最小值（对应基频周期），通过阈值排除虚假峰值。  

**优点**：  
- 计算高效，实时性好。  
- 对倍频错误鲁棒性较强。  

**缺点**：  
- 固定阈值可能不适应复杂信号（如高噪声或快速变化的基频）。  
- 对清浊音边界判断不够精细。  

### 频域方法：

- 倒谱分析(`Cepstrum`)：对信号傅里叶变换后的对数功率谱再作傅里叶逆变换，提取倒谱中的峰值对应基频。
- 谐波积谱乘积谱(`Harmonic Product Spectrum`)
- 频谱峰值检测

### 混合方法：

- `RAPT`算法
- `PRAAT`使用的算法: 结合自相关和倒谱分析，适用于多种语音类型。
- `PYIN`：`YIN`的改进版，结合`HMM`提升鲁棒性。
- `CREPE`：基于`CNN`的深度学习模型，高精度但需`GPU`。

## 应用场景

### 语音识别：

- 声调语言(如中文)的音调识别
- 区分疑问句和陈述句

### 说话人识别：

- 作为声纹特征之一
- 反映说话人生理特征

### 语音合成：

- 控制合成语音的语调
- 情感表达的重要参数

### 病理语音分析：

- 检测声带异常
- 评估发音障碍

### 情感识别：

- 愤怒时基频通常升高且变化大
- 悲伤时基频通常降低且变化小

## 基频分析中的挑战

### 浊音/清音判断：

- 清音段(如/s/, /f/)没有周期性，基频无意义

### 倍频/半频错误：

- 算法可能检测到真实基频的整数倍或分数

### 噪声影响：

- 背景噪声会干扰基频检测

### 实时性要求：

- 某些应用需要低延迟的基频估计

基频特征是语音分析的基础，但通常需要与其他特征(如MFCC、共振峰等)结合使用，才能全面描述语音特性。
好的，抛物线插值（Parabolic Interpolation）用于精确峰值定位是一种非常经典且高效的方法。它通过拟合抛物线来逼近离散采样数据的峰值点，从而获得**亚像素级（Sub-bin）** 的精确定位，克服了FFT等频谱分析中因离散化带来的频率、幅度定位误差。

### 核心思想

在离散频谱（如FFT幅度谱）中，真正的峰值往往位于两个采样点之间。我们无法直接采样到它，但可以通过峰值点及其左右邻居这三个点来拟合一条抛物线，这条抛物线的顶点就是我们对真实峰值的更精确估计。

---

### 数学模型

假设我们有三个相邻的离散数据点，它们构成了一个峰值：
*   `(x₀, y₀)`: 峰值点左边的点
*   `(x₁, y₁)`: 找到的峰值点（幅度最大）
*   `(x₂, y₂)`: 峰值点右边的点

我们的目标是找到一条经过这三点的抛物线 $y = a(x - p)^2 + b$ 的顶点坐标 $(p, b)$。

由于采样点通常是等间距的，我们可以设：
*   $x_0 = -1$, $y_0 = y_[n-1]$
*   $x_1 = 0$, $y_1 = y_[n]$
*   $x_2 = 1$, $y_2 = y_[n+1]$

将这三点代入抛物线方程 $y = ax^2 + bx + c$，得到方程组：
$$
\begin{cases}
y_0 = a(-1)^2 + b(-1) + c = a - b + c \\
y_1 = a(0)^2 + b(0) + c = c \\
y_2 = a(1)^2 + b(1) + c = a + b + c \\
\end{cases}
$$

### 推导公式

通过解这个方程组，可以推导出顶点偏移量 $p$ 和修正后的峰值高度 $b$（顶点处的y值）。

1.  **顶点位置偏移量 (p)**:
    $$
    p = \frac{y_0 - y_2}{2(y_0 - 2y_1 + y_2)}
    $$
    *   `p` 是一个在区间 `(-0.5, 0.5)` 内的偏移量。
    *   **如果 `p > 0`**，表示真实峰值在 bin `n` 的**右侧**。
    *   **如果 `p < 0`**，表示真实峰值在 bin `n` 的**左侧**。

2.  **修正后的峰值频率 (Frequency)**:
$$
    ` corrected\_bin = n + p `
    ` corrected\_frequency = (corrected\_bin) * \frac{sample\_rate}{fft\_size} `
$$
3.  **修正后的峰值幅度 (Amplitude)**:
    $$
    y_{peak} = y_1 - \frac{p}{2}(y_0 - y_2)
    $$
    或者由顶点公式 $y_{peak} = c - \frac{b^2}{4a}$ 推导出的另一个等价形式：
    $$
    y_{peak} = y_1 + \frac{p}{2}(y_0 - y_2)
    $$
    *(注意符号，取决于方程组设定，但结果是等价的)*。更常用的形式是：
    $$
    y_{peak} = y_1 - \frac{1}{4}(y_0 - y_2)p
    $$

### Rust 代码实现

下面是一个完整的 Rust 函数实现，用于抛物线插值精确定位：

```rust
/// 使用抛物线插值对离散频谱峰值进行精确定位
///
/// # 参数
/// * `left`: 峰值左侧点的幅度 (bin n-1)
/// * `center`: 峰值点的幅度 (bin n)
/// * `right`: 峰值右侧点的幅度 (bin n+1)
///
/// # 返回
/// 返回一个元组 `(bin_offset, peak_amplitude)`:
/// * `bin_offset`: 相对于中心点 bin n 的偏移量。例如，0.25 表示真实峰值在 n + 0.25 处。
/// * `peak_amplitude`: 插值后估计的真实峰值幅度。
pub fn parabolic_interpolation(left: f32, center: f32, right: f32) -> (f32, f32) {
    // 计算相对于中心点的偏移量 p
    let numerator = left - right;
    let denominator = left - 2.0 * center + right;
    
    // 避免除以零，如果分母接近零，说明峰值很平坦，偏移量约为0。
    if denominator.abs() < 1e-10 {
        return (0.0, center);
    }
    
    let p = numerator / (2.0 * denominator);
    
    // 计算插值后的峰值幅度
    let peak_amplitude = center - 0.25 * (left - right) * p;
    // 另一种等价计算方式：
    // let peak_amplitude = center - p * (left - right) * 0.5;
    
    (p, peak_amplitude)
}

/// 辅助函数：计算修正后的频率
///
/// # 参数
/// * `peak_bin`: 初始估计的峰值bin索引 (整数)
/// * `bin_offset`: 由 `parabolic_interpolation` 计算出的偏移量
/// * `sample_rate`: 采样率 (Hz)
/// * `fft_size`: FFT 的大小
///
/// # 返回
/// 修正后的精确频率 (Hz)
pub fn calculate_corrected_frequency(
    peak_bin: usize,
    bin_offset: f32,
    sample_rate: u32,
    fft_size: usize,
) -> f32 {
    let corrected_bin = peak_bin as f32 + bin_offset;
    corrected_bin * (sample_rate as f32 / fft_size as f32)
}
```

### 使用示例

假设我们进行了一次 FFT，并在幅度谱中找到了一个峰值位于索引 `idx` 处。

```rust
fn main() {
    // 假设的 FFT 幅度谱（仅展示峰值附近）
    let spectrum: Vec<f32> = vec![...]; // 你的频谱数据
    let sample_rate = 44100; // 采样率 44.1 kHz
    let fft_size = 1024; // FFT 大小

    // 1. 找到最大值的索引（粗略峰值位置）
    let peak_index = find_peak_index(&spectrum); // 假设这个函数已实现

    // 2. 确保峰值不在边界上，否则无法插值
    if peak_index > 0 && peak_index < spectrum.len() - 1 {
        let left = spectrum[peak_index - 1];
        let center = spectrum[peak_index];
        let right = spectrum[peak_index + 1];

        // 3. 进行抛物线插值
        let (bin_offset, corrected_amplitude) = parabolic_interpolation(left, center, right);

        // 4. 计算精确频率
        let exact_freq = calculate_corrected_frequency(peak_index, bin_offset, sample_rate, fft_size);

        println!("粗略峰值: bin {}, ~{:.2} Hz", peak_index, peak_index as f32 * sample_rate as f32 / fft_size as f32);
        println!("精确峰值: bin {:.3}, {:.2} Hz, 幅度 {:.3}", peak_index as f32 + bin_offset, exact_freq, corrected_amplitude);
    } else {
        println!("峰值位于边界，无法使用抛物线插值");
    }
}

// 一个简单的查找峰值索引的函数
fn find_peak_index(spectrum: &[f32]) -> usize {
    let mut max_index = 0;
    for (i, &value) in spectrum.iter().enumerate() {
        if value > spectrum[max_index] {
            max_index = i;
        }
    }
    max_index
}
```

### 注意事项

1.  **适用范围**：该方法假设峰值形状**近似于抛物线**。对于非常尖锐或畸变的峰值，效果可能会打折扣。
2.  **边界检查**：**绝对不能**对频谱的第一个或最后一个点使用此方法，因为它们没有左右邻居。
3.  **分母为零**：公式中分母 `2(y₀ - 2y₁ + y₂)` 可能为零，代码中必须进行处理，否则会导致 panic。
4.  **频率分辨率**：这种方法**提高了频率定位精度**，但**并不能提高FFT固有的频率分辨率**。它只是对现有信息进行更好的估计。
5.  **多种应用**：此方法不仅用于频率定位，还可用于任何需要**精确估计离散信号峰值时间和幅度**的场景，例如：Pitch Detection、ODF（Onset Detection Function）峰值定位等。

抛物线插值以其计算量小、效果好的特点，成为了音频信号处理中精确定位峰值的标准技术之一。
